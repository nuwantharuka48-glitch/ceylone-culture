# Ceylone Culture Restaurant Management System - Java Backend

This directory contains the standalone Java REST API server. It is built using Java's built-in `com.sun.net.httpserver` library, making it completely **zero-dependency** and extremely easy to compile and execute without needing Gradle or Maven installed.

## Prerequisites
To run the backend, you need the **Java Development Kit (JDK)** installed on your machine.
- Verify Java installation by running:
  ```powershell
  java -version
  javac -version
  ```

---

## How to Compile & Run

Open your command prompt or PowerShell and navigate to the project root directory (`C:\Users\MSI\.gemini\antigravity\scratch\ceylone-culture`):

### 1. Compile the Java Class
Run the compiler from the project root:
```powershell
javac backend/CeyloneCultureServer.java
```
This compiles the code and creates `backend/CeyloneCultureServer.class`.

### 2. Launch the REST Server
Run the compiled bytecode class:
```powershell
java backend.CeyloneCultureServer
```

You should see output similar to this:
```
=================================================
   CEYLONE CULTURE - RESTAURANT BACKEND REST API 
=================================================
Server is starting up...
Running on: http://localhost:8080
Endpoints available:
  - GET  /api/status
  - GET  /api/orders      | POST /api/orders
  - POST /api/orders/status
  - GET  /api/bulk-orders | POST /api/bulk-orders
  - GET  /api/promotions  | POST /api/promotions
Press Ctrl+C to stop the server.
=================================================
```

---

## Connecting the Frontend

Once the Java server is running:
1. Open `frontend/index.html` in your web browser.
2. Look at the bottom-left sidebar. The status dot will instantly turn **green** and say **"Connected to Java Backend"**.
3. Any order placed, event registered, or promotion coupon deployed will now be sent directly to the Java server's in-memory storage. 
4. If you refresh the page, the frontend will automatically load and synchronize the state directly from your running Java server!

## In-Memory Database API Mappings
- **`GET /api/status`**: Check server viability.
- **`GET /api/orders` & `POST /api/orders`**: Retrieve and append standard dine-in or takeaway orders.
- **`POST /api/orders/status`**: Advance order statuses through their lifecycle (`Preparing` -> `Ready` -> `Served`).
- **`GET /api/bulk-orders` & `POST /api/bulk-orders`**: Retrieve and record large corporate events and catering package logs.
- **`GET /api/promotions` & `POST /api/promotions`**: Retrieve and deploy active campaign promo codes generated by the AI engine.
